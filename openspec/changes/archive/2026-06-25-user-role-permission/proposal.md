## Why

目前系统缺乏用户角色区分，所有用户都共享相同权限。无法控制不同用户对不同机器人的访问权限，也无法区分管理员与普通用户。需要建立角色权限体系，实现精细化的机器人访问控制。

## What Changes

- 引入三种用户角色：匿名用户、内部用户、管理员
- 管理员可访问管理后台，普通用户和匿名用户不能访问
- 匿名用户只能使用默认机器人创建对话
- 内部用户可选择已授权的机器人创建对话
- 管理员可在后台配置每个机器人的访问权限（所有人/指定用户/指定团队）
- 团队概念：内部用户可属于某个团队

## Capabilities

### New Capabilities

- **user-roles**: 用户角色系统，区分匿名用户、内部用户、管理员
- **bot-access-control**: 机器人访问权限控制，支持按用户或团队授权
- **team-management**: 团队管理，支持创建团队、添加/移除团队成员

### Modified Capabilities

- (无)

## Impact

- **数据库**: 新增 User 表（存储内部用户）、Team 表、BotAccess 表
- **后端**: 新增角色验证中间件、访问控制逻辑、团队管理 API
- **前端**: 登录页面区分内部用户登录、管理后台需权限验证、机器人选择器只显示有权限的机器人
- **现有功能**: 会话列表功能需与用户角色配合（匿名用户只看当前会话）
