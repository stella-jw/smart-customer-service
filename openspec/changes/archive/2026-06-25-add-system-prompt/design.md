## Context

当前 `BotConfiguration` 表只包含 `personality` 和 `response_tone` 两个有限的行为参数。LLM 调用时没有传入系统级提示词，导致无法为不同行业的机器人提供专属的角色设定和回答逻辑。

## Goals / Non-Goals

**Goals:**
- 为每个机器人支持自定义系统提示词
- 系统提示词在 LLM 调用时作为 `system` 参数传入
- 在管理界面提供便捷的编辑体验

**Non-Goals:**
- 不支持动态变量替换（如 {user_name}）
- 不支持多轮对话记忆优化

## Decisions

### 1. 存储方式

**方案**: 在 `BotConfiguration` 表添加 `system_prompt` 字段（Text 类型）

**备选方案**: 独立的 `SystemPrompt` 表
- 增加复杂性，当前场景不需要

**结论**: 直接在 BotConfiguration 中添加字段

### 2. LLM 调用集成

**方案**: 在 LangGraph 的 generate 节点接收 `system_prompt` 参数

**备选方案**: 在 API 层拼装好再传给 LLM
- 违背 LangGraph 的设计原则

**结论**: 在 generate 节点中从 bot_config 读取并传入

### 3. 默认提示词

**方案**: 提供基于行业类型和人格设定的默认提示词模板

- professional: "你是一个专业严谨的智能客服助手，回答问题时条理清晰、言简意赅。你的目标是高效准确地解决用户问题。"

**结论**: 新建机器人时自动填充默认提示词，允许用户自定义修改

## Risks / Trade-offs

- **风险**: 用户填写的提示词过长导致 LLM 处理变慢
- **处理**: 限制提示词不超过 1200 字符

- **风险**: 用户输入特殊字符或代码可能破坏 prompt 结构
- **处理**: 验证只允许中文、英文、数字、常用标点符号，以及 email 和电话号码格式

## Content Validation

**允许的字符**：
- 中文文字（\u4e00-\u9fff）
- 英文字母（a-zA-Z）
- 数字（0-9）
- 常用标点：，。！？；：""''（）【】《》、·-
- 空格和换行
- email 格式：xxx@xxx.xxx
- 电话号码格式：+86 或 0 开头，数字和横杠

**禁止的字符**：
- 代码类：\`\`\`, \`\`, \${, \{\{, \{\%
- 特殊符号：@（email外）, #, &, *, |, \\, /, <, >, =, !, ~, ^, 等
- 控制字符：\\n\\r\\t 以外的不可见字符

- **风险**: 提示词配置错误导致 LLM 回复质量下降
- **处理**: 提供默认模板，用户可参考修改
