## Why

当前 Admin 前端硬编码使用单个 Bot，无法支持多机器人场景。实际业务需要多个独立机器人：如外部客服机器人（面向客户，RAG 产品手册）和内部 IT 助手（面向员工，RAG 员工手册）。

同时需要调整 Admin 页面布局为左右分栏结构，提升多机器人场景下的操作效率。

## What Changes

1. **页面布局调整** - 左右分栏：左侧导航菜单，右侧内容区域
2. **机器人管理** - Bot 列表增删改查、每个 Bot 独立统计（对话数、满意度）
3. **Bot 数量限制** - 最多 5 个机器人，达到上限禁用添加按钮
4. **知识库管理** - 文档明确归属某个机器人，支持按机器人筛选
5. **Bot 配置** - 每个 Bot 独立配置 RAG 和 QA 参数

## Capabilities

### New Capabilities

- `bot-management`: 多机器人管理界面（列表、创建、删除、数量限制、统计）
- `knowledge-base-multi-bot`: 知识库按机器人隔离管理
- `admin-layout`: 左右分栏管理后台布局

### Modified Capabilities

- `qa-management`: 支持按机器人筛选 QA 对

## Impact

- **前端**: Admin 端全新布局（左侧导航 + 右侧内容）、机器人管理页、知识库管理页
- **后端**: API 已支持多机器人，只需前端调用时传递 bot_id
- **用户端**: 用户可选择不同的机器人入口（后续实现）
