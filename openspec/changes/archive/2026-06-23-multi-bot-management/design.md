## Context

当前 Admin 前端所有 API 调用硬编码 `BOT_ID = 'default-bot'`，无法切换机器人。需要实现完整的多机器人管理功能，包括机器人管理、知识库管理（按机器人隔离）、QA管理等，并调整页面布局为左右分栏结构。

## Goals / Non-Goals

**Goals:**
- 实现 Bot 列表管理页面（增删改查）
- 限制 Bot 数量最多 5 个
- 每个 Bot 独立配置 RAG 和 QA
- 每个 Bot 有独立的对话数量和满意度统计
- 知识库文档明确归属于某个 Bot
- 左右分栏布局：左侧导航，右侧内容

**Non-Goals:**
- 不实现 Bot 之间的数据迁移
- 不实现 Bot 克隆功能
- 不实现知识库文档编辑（只支持增删查）
- 不实现多语言支持

## Decisions

### Decision 1: 页面布局

**选择**: 左右分栏布局

```
┌──────────────────────────────────────────────────────┐
│  智能客服系统                      [机器人切换] [退出] │
├────────────┬─────────────────────────────────────────┤
│            │                                         │
│  机器人管理  │          右侧内容区域                    │
│  知识库     │                                         │
│  QA管理     │                                         │
│  机器人配置  │                                         │
│            │                                         │
└────────────┴─────────────────────────────────────────┘
```

**理由**: 导航清晰，各模块入口明确，适合多机器人切换场景。

### Decision 2: Bot 数量限制方案

**选择**: 前端限制 + 后端验证

```
前端: 达到5个后"添加Bot"按钮禁用
后端: /api/admin/bots POST 返回 400 "已达上限"
```

**理由**: 双保险，前端改善 UX，后端保证安全。

### Decision 3: Bot 切换状态管理

**选择**: Pinia Store 集中管理

```typescript
// stores/botStore.ts
const currentBotId = ref<string>(localStorage.getItem('current_bot_id') || '')
const botList = ref<Bot[]>([])

// 切换时保存到 localStorage
watch(currentBotId, (newId) => localStorage.setItem('current_bot_id', newId))
```

**理由**: 集中管理、刷新不丢失、多组件共享。

### Decision 4: 知识库文档归属

**选择**: 文档表 `bot_id` 明确关联

```
文档列表显示:
┌────────────────────────────────────────────┐
│ 文档名称        │ 所属机器人   │ 状态     │
├────────────────────────────────────────────┤
│ 产品手册.pdf     │ 电商客服Bot  │ 已索引   │
│ 常见问题.md      │ 电商客服Bot  │ 已索引   │
│ 技术文档.docx    │ IT支持Bot    │ 处理中   │
└────────────────────────────────────────────┘
```

**理由**: 文档归属清晰，便于管理。

### Decision 5: Bot 统计信息

每个 Bot 卡片显示:
- 对话总数
- 今日对话
- 平均满意度
- RAG 命中率
- QA 匹配率

## Risks / Trade-offs

| 风险 | 影响 | 缓解 |
|------|------|------|
| Bot 删除后数据残留 | ChromaDB collection 仍存在 | 删除时同步清理 ChromaDB |
| 切换 Bot 页面数据未刷新 | 用户看到旧数据 | 切换后强制刷新当前页面数据 |
| 文档归属错误 | 文档进入错误机器人 | 上传时必须选择目标机器人 |

## Migration Plan

1. 新增 AdminLayout 左右分栏布局
2. 新增 BotStore 状态管理
3. 新增 BotSwitcher 组件
4. 重构 BotsPage 机器人管理页
5. 重构 KnowledgeBasePage 支持按机器人筛选
6. 重构 QA/BotConfig 页面使用当前 Bot
